# Example Requirements - Customer Churn Prediction Model

## Project Overview
Build a predictive model to identify customers at risk of churning from a subscription or loyalty program.

## Objectives
1. Predict customer churn probability with 85%+ accuracy
2. Identify key features driving churn behavior
3. Provide actionable insights for retention strategies

## Data Requirements
- Customer transaction history (last 12 months)
- Rewards program engagement metrics
- Demographic information
- Customer lifetime value (CLV)
- Historical churn labels

## Model Requirements
- Type: Binary classification (churn vs. non-churn)
- Algorithm: Compare Random Forest, XGBoost, Logistic Regression
- Validation: 5-fold cross-validation
- Metrics: Precision, Recall, F1-Score, AUC-ROC

## Deliverables
1. Technical specification document
2. Model training pipeline
3. Feature importance analysis
4. Model performance report
5. Deployment recommendations

## Timeline
- Specification: Week 1
- Data preparation: Week 2
- Model development: Weeks 3-4
- Validation and reporting: Week 5

## Success Criteria
- Model AUC-ROC > 0.85
- Top 20% predicted churners contain >60% of actual churners
- Feature importance aligns with business intuition
- Model explainability for non-technical stakeholders
